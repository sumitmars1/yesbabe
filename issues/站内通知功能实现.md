# 站内通知功能实现

## 任务描述

实现站内通知功能，具体要求如下：

1. **WebSocket连接实现**：
   - 建立持久化WebSocket连接专用于站内通知功能
   - 连接权限控制：仅允许已认证用户建立连接
   - 用户登录后需要连接
   - 连接稳定性保障：
     * 用户刷新页面后自动重新建立连接
     * 连接异常断开后自动尝试重连
   - 连接地址规范：ws://47.94.83.43:8000/api/ws/notification?token=exxx
   - 参数规范：必须与聊天WebSocket保持一致的连接参数

2. **通知展示组件**：
   - 使用NaiveUI的Notification组件实现通知展示功能
   - 组件需按照官方文档规范进行配置和使用

## 实现方案

### 架构设计

选择**独立通知系统 + 事件驱动模式**：

1. **独立的NotificationWebSocketManager类**：
   - 基于现有WebSocketManager架构
   - 专门处理通知业务逻辑
   - 与聊天WebSocket完全解耦

2. **事件驱动模式**：
   - 解耦WebSocket连接和UI展示
   - 使用composable模式管理状态
   - 直接集成NaiveUI Notification API

## 技术细节

### 连接参数处理

根据`VUE_CHANNEL_TRACKING_INTEGRATION.md`文档，WebSocket连接需要包含以下参数：

- `token`: 用户认证令牌
- `user_agent`: 浏览器User-Agent
- `accept_language`: 语言偏好
- `referer`: 当前页面URL
- `x_fingerprint`: JSON格式的设备指纹信息
- `channel_code`: 渠道追踪参数（可选）

### 通知消息格式

#### 普通通知消息
```typescript
interface NotificationMessage {
  type: 'notification';
  data: {
    id: string;
    title: string;
    content: string;
    notificationType: 'info' | 'success' | 'warning' | 'error';
    timestamp: string;
    duration?: number;
    action?: {
      text: string;
      handler: () => void;
    };
  };
}
```

#### 未读消息通知（服务器响应格式）
```typescript
interface NotificationServerResponse {
  code: number;
  message: string;
  data: {
    type: 'unread_notifications';
    notifications: Array<{
      id: number;
      title: string;
      content: string;
      type: 'system' | 'user' | 'admin';
      status: 'unread' | 'read';
      created_at: string;
      read_at: string | null;
      additional_data: any;
    }>;
    total: number;
  };
}
```

### 自动连接机制

- 监听`authStore.isLoggedIn`状态变化
- 用户登录时自动建立连接
- 用户登出时自动断开连接
- 页面刷新后根据登录状态自动重连

### 重连策略

- 指数退避算法：1s, 2s, 4s, 8s, 16s, 30s（最大）
- 最大重连次数：10次
- 网络错误和认证失败的不同处理策略

## 使用方式

### 在组件中获取通知状态

```vue
<script setup>
import { useNotificationWebSocket } from '@/composables/useNotificationWebSocket';

const {
  isConnected,
  isConnecting,
  connectionStatus,
  error,
  lastNotification
} = useNotificationWebSocket();
</script>
```

### 通知显示

通知会自动通过NaiveUI的Notification组件显示，支持：
- 不同类型的样式（success, warning, error, info）
- 5秒自动关闭
- 鼠标悬停时保持显示
- 自定义标题和内容

## 注意事项

1. **权限控制**：只有已认证用户才能建立通知WebSocket连接
2. **资源管理**：组件卸载时自动清理WebSocket连接
3. **错误处理**：连接失败时提供详细的错误信息
4. **性能优化**：使用computed和响应式状态减少不必要的重渲染
5. **兼容性**：复用现有的指纹生成和渠道追踪逻辑
