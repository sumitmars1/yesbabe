# 聊天内容高度变化自动滚动实现

## 任务概述

实现监听聊天内容高度变化，自动滚动到底部的功能，解决现有监听机制可能遗漏某些内容变化的问题。

## 解决方案

### 实现思路

使用 `ResizeObserver` API 监听消息容器的尺寸变化，当内容高度发生变化时，自动滚动到底部。同时添加节流控制，避免频繁滚动影响性能。

### 主要变更

1. 创建通用的节流函数 `throttle.ts`
2. 在 `Chat.vue` 中添加 `ResizeObserver` 监听逻辑
3. 保留现有的消息监听逻辑作为备份机制

## 实施步骤

### 步骤1：创建节流函数

✅ 创建 `src/utils/throttle.ts` 文件，实现通用的节流函数

### 步骤2：修改 Chat.vue 组件

✅ 主要变更：
- 添加消息容器的引用 `messageContainerRef`
- 在 `onMounted` 中初始化 `ResizeObserver` 和节流函数
- 在 `onUnmounted` 中清理 `ResizeObserver` 和节流函数
- 保留现有的消息监听逻辑作为备份机制

## 文件变更记录

### 新增文件
- `src/utils/throttle.ts` - 通用节流函数

### 修改文件
- `src/views/Chat/Component/Chat/Chat.vue` - 添加 ResizeObserver 监听逻辑

## 注意事项

1. 使用节流控制，限制 1 秒内只触发一次滚动，避免频繁滚动影响性能
2. 保留现有的消息监听逻辑作为备份机制，确保兼容性
3. 在组件卸载时清理 ResizeObserver，避免内存泄漏
