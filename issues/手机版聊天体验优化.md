# 手机版聊天体验优化

## 任务概述
优化移动端聊天界面的用户体验，重点解决键盘弹出时的界面适应、滚动逻辑优化和发送消息后的自动滚动问题，参考微信聊天体验标准。

## 问题分析

### 现有问题
1. **滚动逻辑不够智能**：移动端滚动到底部的逻辑不够精确
2. **键盘弹出处理**：输入框获得焦点时，键盘弹出可能遮挡聊天内容
3. **发送消息后滚动**：发送消息后滚动时机和效果不够流畅
4. **视口高度变化**：iOS Safari地址栏变化和Android键盘弹出时的视口处理

### 技术挑战
- 不同移动端浏览器的键盘行为差异
- Visual Viewport API的兼容性处理
- 滚动时机的精确控制
- 动态视口高度的计算

## 解决方案

### 方案选择
采用**方案二**（专用移动端组合函数）结合**方案一**（渐进式优化）的混合策略：
- 创建专门的`useMobileChatExperience`组合函数
- 保持现有架构稳定性
- 渐进式集成新功能

### 核心功能设计
1. **键盘检测机制**
   - 优先使用Visual Viewport API
   - 降级使用window.innerHeight变化检测
   - 支持iOS Safari和Android Chrome

2. **视口管理**
   - 动态计算可用视口高度
   - 处理键盘弹出时的布局调整
   - 管理CSS变量实现响应式高度

3. **滚动优化**
   - 强制滚动到底部的专用方法
   - 多重降级策略确保兼容性
   - 精确的滚动时机控制

### 滚动策略
1. **多重降级机制**
   - 优先使用组件的scrollToBottom方法
   - 降级使用原生scrollTo API
   - 最后使用scrollTop属性

2. **时机控制**
   - 发送消息：立即滚动 + 延迟滚动
   - 消息变化：nextTick后滚动
   - 键盘弹出：延迟滚动（iOS 300ms，Android 100ms）

### 兼容性处理
- **iOS Safari**：处理地址栏变化，使用较长延迟
- **Android Chrome**：优化触摸滚动，使用较短延迟
- **降级支持**：不支持Visual Viewport API时使用resize事件

## 问题修复记录

### 2024-01-XX 用户反馈问题修复

**问题1：JSON格式消息显示异常**
- 现象：用户输入包含JSON格式的文本（如`{"type":"generate_image_to_video"}`）无法正常显示
- 原因：ChatItem.vue中的过滤条件过于严格，误将用户输入的JSON文本当作系统消息过滤
- 解决方案：
  - 修改第2行过滤条件：`!(chat.content === 'generate_video_message' && !chat.isSelf)`
  - 修改第129行过滤条件：同样逻辑，确保用户消息不被过滤
  - 只过滤非用户发送的系统消息，保留用户输入的所有内容

**问题2：Safari移动端输入框点击整体上抬**
- 现象：在Safari浏览器中点击输入框时，整个页面会向上移动
- 原因：Safari的默认行为会调整视口以确保输入框可见
- 解决方案：
  - 添加CSS样式：`body.keyboard-visible { position: fixed; }`
  - 在键盘状态变化时动态添加/移除body类
  - 确保只有内容区域调整，顶部保持固定
  - 在组件卸载时清理body类，防止状态残留
