# 聊天页面滚动逻辑重构任务

## 任务概述
重构 `Chat.vue` 页面的滚动逻辑，解决现有滚动系统存在的问题，并针对不同设备和浏览器进行优化。

## 问题分析
### 原有滚动逻辑存在的问题：
1. **代码复杂度高**：多个滚动函数（`scrollToBottom`、`ensureScrollToBottom`、`delayedScrollToBottom`）职责重叠
2. **setTimeout滥用**：大量使用定时器导致时序问题和性能问题
3. **移动端兼容性差**：iOS Safari和Android浏览器的滚动行为不一致
4. **逻辑混杂**：PC端和移动端逻辑混合在一起，难以维护
5. **重复触发**：缺乏防抖机制，可能导致频繁的滚动调用
6. **缺乏错误处理**：滚动失败时没有完善的降级方案

## 解决方案
### 1. 架构重构
- 创建独立的设备检测组合函数（`useDeviceDetection.ts`）
- 创建统一的滚动管理组合函数（`useScrollManager.ts`）
- 重构 `Chat.vue` 使用新的滚动系统

### 2. 设备检测优化
- 精确检测iOS Safari、Android Chrome等不同浏览器
- 提供设备类型和推荐滚动策略
- 检测现代API支持情况

### 3. 滚动策略分离
- **PC端**：使用Naive UI的原生`scrollTo`方法
- **iOS Safari**：使用`-webkit-overflow-scrolling: touch`和防bounce机制
- **Android**：使用标准`scrollTo`和`overscroll-behavior`控制

### 4. 性能优化
- 实现防抖机制，避免频繁滚动调用
- 检查是否已在底部，避免不必要的滚动
- 使用`requestAnimationFrame`优化动画性能
- 完善的错误处理和降级方案

## 技术要点

### 设备检测
```typescript
// 检测iOS Safari
const isIOSSafari = computed(() => {
  return isIOS.value && /Safari/.test(userAgent) && !/CriOS|FxiOS/.test(userAgent);
});

// 检测Android Chrome
const isAndroidChrome = computed(() => {
  return isAndroid.value && /Chrome/.test(userAgent) && !/Edge/.test(userAgent);
});
```

### 滚动策略
```typescript
// 根据设备选择滚动策略
switch (scrollStrategy.value) {
  case 'ios-webkit':
    await scrollToBottomIOS(container, smooth);
    break;
  case 'android-standard':
    await scrollToBottomAndroid(container, smooth);
    break;
  case 'desktop':
    await scrollToBottomDesktop(scrollbarRef, smooth);
    break;
}
```

### CSS优化
```css
/* iOS Safari 专用优化 */
@supports (-webkit-touch-callout: none) {
  .mobile-scroll-container {
    -webkit-overflow-scrolling: touch;
    overscroll-behavior: contain;
  }
}

/* Android Chrome 专用优化 */
@media screen and (-webkit-min-device-pixel-ratio: 0) {
  .mobile-scroll-container {
    overscroll-behavior: contain;
  }
}
```
