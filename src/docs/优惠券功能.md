## 1. 概述

### 1.1 优惠券系统架构

```
用户注册 ──▶ 自动发券（claim_type=auto）
                                            ┌───────────────┐
用户在券中心浏览 ──▶ 手动领取 ──────────────▶│               │
                                            │  我的优惠券    │
用户输入兑换码 ──▶ 兑换码领取 ──────────────▶│  (券包)        │
                                            └──────┬────────┘
                                                   │
                                                   ▼
                              下单页面 ──▶ 查询可用券（含折扣预览）
                                                   │
                                                   ▼
                              提交订单 ──▶ 传入 coupon_id ──▶ 后端自动抵扣
```

### 1.2 优惠券类型

| 折扣类型 | discount_type | 说明 | 示例 |
|----------|--------------|------|------|
| 固定金额 | `fixed` | 直接减免固定金额 | 减 50,000₫ |
| 百分比折扣 | `percentage` | 按订单比例折扣 | 打 9 折（减 10%），封顶 100,000₫ |

### 1.3 领取方式

| 领取方式 | claim_type | 前端行为 |
|----------|-----------|----------|
| 自动发放 | `auto` | **无需前端操作** — 用户注册后自动到账 |
| 手动领取 | `manual` | 用户在「券中心」点击「领取」按钮 |
| 兑换码 | `code` | 用户输入兑换码文本框，点击「兑换」 |

### 1.4 优惠券状态

| 状态 | status | 说明 |
|------|--------|------|
| 可用 | `available` | 可在下单时选择使用 |
| 已使用 | `used` | 已核销，关联了订单 |
| 已过期 | `expired` | 超过有效期 |
| 已取消 | `cancelled` | 被管理员手动取消 |

```http
Authorization: Bearer <access_token>
```

## 3. API 接口一览

所有接口的 Base URL 为 `/api/pay/coupon`。

| 接口 | 方法 | 路径 | 用途 | 限频 |
|------|------|------|------|------|
| 可领取列表 | GET | `/available` | 浏览可以领的券 | 通用限频 |
| 手动领取 | POST | `/claim` | 按模板ID领取 | 支付限频 |
| 兑换码领取 | POST | `/redeem` | 输入兑换码兑换 | 支付限频 |
| 我的优惠券 | GET | `/mine` | 查看券包 | 通用限频 |
| 订单可用券 | GET | `/applicable` | 下单时查询可用券 | 通用限频 |

### 4.1 获取可领取的优惠券

浏览当前用户可以领取的优惠券列表。已按用户语言（地区）自动过滤，已达领取上限的模板不会返回。

**请求**

```http
GET /api/pay/coupon/available
Authorization: Bearer <token>
```

**响应**

```json
{
  "code": 200,
  "message": "success",
  "data": [
    {
      "template_id": 1,
      "name": "新用户首单8折",
      "description": "新用户专享，订阅VIP立享8折优惠",
      "discount_type": "percentage",
      "discount_value": "20",
      "max_discount": "100000",
      "currency": null,
      "min_amount": "0",
      "applicable_type": "vip",
      "validity_type": "days_after_claim",
      "validity_days": 7,
      "valid_start": null,
      "valid_end": null,
      "remaining": 980
    },
    {
      "template_id": 2,
      "name": "越南春节满减",
      "description": "满200,000₫减50,000₫",
      "discount_type": "fixed",
      "discount_value": "50000",
      "max_discount": null,
      "currency": "VND",
      "min_amount": "200000",
      "applicable_type": "all",
      "validity_type": "fixed_date",
      "validity_days": null,
      "valid_start": "2026-02-01T00:00:00",
      "valid_end": "2026-02-28T23:59:59",
      "remaining": -1
    }
  ]
}
```

**字段说明**

| 字段 | 类型 | 说明 |
|------|------|------|
| `template_id` | int | 模板ID，领取时需要传此ID |
| `name` | string | 优惠券名称，直接展示给用户 |
| `description` | string? | 优惠券描述文字 |
| `discount_type` | string | `fixed` = 固定金额, `percentage` = 百分比 |
| `discount_value` | string | 折扣值。fixed 时为金额；percentage 时为百分比数字 |
| `max_discount` | string? | 百分比折扣的封顶金额，fixed 类型为 null |
| `currency` | string? | 适用货币（如 `VND`），百分比类型可能为 null |
| `min_amount` | string | 最低消费门槛，`0` = 无门槛 |
| `applicable_type` | string | `all` = 全部, `vip` = 仅VIP, `token` = 仅钻石充值 |
| `validity_type` | string | `fixed_date` = 固定日期, `days_after_claim` = 领取后N天 |
| `validity_days` | int? | 领取后有效天数（仅 `days_after_claim` 类型） |
| `valid_start` / `valid_end` | string? | 固定生效起止时间（仅 `fixed_date` 类型） |
| `remaining` | int? | 剩余可领数量，`-1` = 不限量 |

### 4.2 手动领取优惠券

用户在「券中心」点击「领取」按钮时调用。

**请求**

```http
POST /api/pay/coupon/claim
Authorization: Bearer <token>
Content-Type: application/json

{
  "template_id": 1
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `template_id` | int | 是 | 来自 `/available` 接口返回的 `template_id` |

**成功响应**

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "coupon_id": 42,
    "valid_from": "2026-02-10T12:00:00",
    "valid_until": "2026-02-17T12:00:00"
  }
}
```

| 字段 | 说明 |
|------|------|
| `coupon_id` | 用户优惠券实例ID，后续下单时传此ID |
| `valid_from` | 生效时间 |
| `valid_until` | 过期时间 |

**失败情况**

| 场景 | HTTP 状态码 | 错误信息 |
|------|------------|----------|
| 领取处理中（并发） | 400 | Coupon claim is being processed, please try again |
| 券已领完 | 400 | This coupon has been fully claimed |
| 已达领取上限 | 400 | You have reached the claim limit for this coupon |
| 模板不存在/已停用 | 400 | Coupon not available |
| 地区不匹配 | 400 | This coupon is not available in your region |

### 4.3 兑换码领取

用户输入兑换码文本，点击「兑换」按钮。

**请求**

```http
POST /api/pay/coupon/redeem
Authorization: Bearer <token>
Content-Type: application/json

{
  "code": "NEWUSER2026"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `code` | string | 是 | 兑换码，1-32 个字符 |

**成功响应**

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "coupon_id": 43,
    "valid_from": "2026-02-10T12:00:00",
    "valid_until": "2026-02-17T12:00:00"
  }
}
```

响应结构与「手动领取」完全一致。

**失败情况**

| 场景 | HTTP 状态码 | 错误信息 |
|------|------------|----------|
| 兑换码不存在 | 404 | Invalid or expired redemption code |
| 兑换码不是 code 类型 | 400 | Invalid redemption code |
| 其余同手动领取 | - | 同上 |

**前端 UI 建议**

### 4.5 查询订单可用券（含折扣预览）

用户在下单页面，选择商品后，调用此接口查询「这笔订单能用哪些券」，并展示每张券能省多少钱。

**请求**

```http
GET /api/pay/coupon/applicable?order_type=vip&product_id=3
Authorization: Bearer <token>
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `order_type` | string | 是 | `vip` 或 `token` |
| `product_id` | int | 是 | 商品ID（VIP套餐ID 或 钻石套餐ID） |

**响应**

```json
{
  "code": 200,
  "message": "success",
  "data": [
    {
      "coupon_id": 42,
      "name": "新用户首单8折",
      "discount_type": "percentage",
      "discount_value": "20",
      "discount_preview": "60000",
      "final_amount_preview": "240000",
      "valid_until": "2026-02-17T12:00:00"
    }
  ]
}
```

**字段说明**

| 字段 | 类型 | 说明 |
|------|------|------|
| `coupon_id` | int | **下单时传这个 ID** |
| `name` | string | 券名称 |
| `discount_type` | string | `fixed` / `percentage` |
| `discount_value` | string | 折扣值 |
| `discount_preview` | string | **预计折扣金额**（已按实际订单金额计算） |
| `final_amount_preview` | string | **预计应付金额**（原价 - 折扣） |
| `valid_until` | string | 过期时间 |

> **重要**：`discount_preview` 和 `final_amount_preview` 是后端根据实际商品价格计算好的，前端直接展示即可，**不要自己算折扣**。

## 5. 支付下单时使用优惠券

选好券后提交支付订单时，在原有支付请求中**新增 `coupon_id` 字段**即可。

### 5.1 VIP 订阅支付

```http
POST /api/pay/vn/vip
Authorization: Bearer <token>
Content-Type: application/json

{
  "vip_id": 3,
  "pay_method": "VNBANKQR",
  "coupon_id": 42,
  "back_url": "https://example.com/pay/result"
}
```

### 5.2 钻石充值支付

```http
POST /api/pay/vn/token
Authorization: Bearer <token>
Content-Type: application/json

{
  "token_id": 5,
  "pay_method": "VNBANKQR",
  "coupon_id": 42,
  "back_url": "https://example.com/pay/result"
}
```

### 5.3 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `coupon_id` | int | **否** | 用户优惠券实例ID（来自 `/applicable` 或 `/mine` 的 `coupon_id`）。不传或传 `null` = 不使用优惠券 |

### 5.4 后端处理流程

```
前端提交 coupon_id
    │
    ▼
后端验证券的状态/有效期/适用性
    │
    ├── 验证失败 → 返回错误（不扣款）
    │
    ├── 折扣后金额 > 0 → 创建订单（实付金额=原价-折扣）→ 跳转支付
    │
    └── 折扣后金额 ≤ 0 → 全额抵扣，直接完成订单（无需支付）
                         返回 provider="coupon_free"
```

### 5.5 全额抵扣（免支付）

当优惠券完全覆盖订单金额时，后端会**直接完成订单**，不跳转支付页面。

响应示例：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "order_id": "VN1707654321",
    "provider": "coupon_free",
    "amount": 300000,
    "final_amount": 0,
    "discount_amount": 300000,
    "status": "completed",
    "pay_url": null,
    "benefits_granted": true
  }
}
```

**前端处理逻辑**：

```javascript
const res = await api.post('/api/pay/vn/vip', payload)
const data = res.data.data

if (data.provider === 'coupon_free') {
  // 全额抵扣，直接跳转成功页
  router.push('/pay/success?order_id=' + data.order_id)
} else if (data.pay_url) {
  // 正常支付，跳转支付页
  window.location.href = data.pay_url
}
```

## 6. 完整业务流程

### 6.1 新用户首次使用优惠券完整流程

```
1. 用户注册
   │
   ▼
2. 后端自动发券（claim_type=auto 的模板按地区匹配）
   │
   ▼
3. 用户进入「VIP订阅」页面，选择套餐（如 vip_id=3）
   │
   ▼
4. 前端调用 GET /api/pay/coupon/applicable?order_type=vip&product_id=3
   │  → 返回可用券列表 + 折扣预览
   │
   ▼
5. 用户选中一张券（如 coupon_id=42，省 ₫60,000）
   │
   ▼
6. 前端提交订单: POST /api/pay/vn/vip  { vip_id: 3, coupon_id: 42, pay_method: "VNBANKQR" }
   │
   ├── 正常: 返回 pay_url → 前端跳转支付
   │
   └── 全额抵扣: provider="coupon_free" → 前端跳转成功页
```

### 6.2 兑换码流程

```
1. 用户获得兑换码（营销活动/客服发放）
   │
   ▼
2. 用户在「券中心」或弹窗输入兑换码
   │
   ▼
3. 前端调用 POST /api/pay/coupon/redeem { code: "SPRING2026" }
   │
   ├── 成功: 展示「领取成功」提示 + 券信息
   │
   └── 失败: 展示错误提示
   │
   ▼
4. 后续正常使用（同 6.1 的步骤 3-6）
```

### 6.3 取消/过期处理

- **用户取消订单**：券自动释放回「可用」状态
- **支付超时**：后端定时任务自动取消超时订单并释放券
- **券到期**：后端定时任务每小时扫描过期券，自动标记为 `expired`

> 前端无需处理释放逻辑，调用 `/mine` 接口时会自动反映最新状态。

## 7. 数据结构参考

### 7.1 优惠券模板（可领取列表返回）

```typescript
interface CouponTemplate {
  template_id: number       // 模板ID
  name: string              // 名称
  description: string | null // 描述
  discount_type: 'fixed' | 'percentage'
  discount_value: string    // 金额或百分比
  max_discount: string | null // 封顶金额
  currency: string | null   // 货币码
  min_amount: string        // 最低消费
  applicable_type: 'all' | 'vip' | 'token'
  validity_type: 'fixed_date' | 'days_after_claim'
  validity_days: number | null
  valid_start: string | null // ISO datetime
  valid_end: string | null
  remaining: number | null  // -1=无限
}
```

### 7.2 用户优惠券（我的券包返回）

```typescript
interface UserCoupon {
  coupon_id: number         // 用户优惠券实例ID（下单时传这个）
  template_id: number
  name: string
  description: string | null
  discount_type: 'fixed' | 'percentage'
  discount_value: string
  max_discount: string | null
  currency: string | null
```

## 9. 最佳实践

### 9.1 下单页优惠券选择推荐实现

```javascript
// 1. 用户选择商品后，立即拉取可用券
async function loadApplicableCoupons(orderType, productId) {
  const res = await api.get('/api/pay/coupon/applicable', {
    params: { order_type: orderType, product_id: productId }
  })
  return res.data.data  // ApplicableCoupon[]
}

// 2. 展示券列表，默认选中折扣最大的
function selectBestCoupon(coupons) {
  if (!coupons.length) return null
  return coupons.reduce((best, c) =>
    parseFloat(c.discount_preview) > parseFloat(best.discount_preview) ? c : best
  )
}

// 3. 提交订单
async function submitOrder(vipId, payMethod, couponId) {
  const payload = {
    vip_id: vipId,
    pay_method: payMethod,
    coupon_id: couponId || undefined  // 不使用券时不传
  }
  const res = await api.post('/api/pay/vn/vip', payload)
  const data = res.data.data

  if (data.provider === 'coupon_free') {
    // 全额抵扣 → 直接成功
    showSuccess('订单已完成（优惠券全额抵扣）')
    router.push('/pay/success')
  } else if (data.pay_url) {
    // 正常支付流程
    window.location.href = data.pay_url
  }
}
```

### 9.2 注意事项

1. **ID 不要搞混**
   - `template_id`：优惠券模板ID — 用于「领取」接口
   - `coupon_id`：用户优惠券实例ID — 用于「下单」和「可用券预览」

2. **金额不要前端自己算**
   - `/applicable` 接口已经返回 `discount_preview` 和 `final_amount_preview`
   - 直接展示后端计算结果，避免前后端计算不一致

3. **防重复点击**
   - 「领取」和「兑换」按钮点击后禁用 + loading 状态
   - 后端有 Redis 锁保护，但前端也应防抖

4. **券状态刷新**
   - 支付完成回到 App 后，重新拉取 `/mine` 刷新券状态
   - 不要本地缓存券状态太久

5. **全额抵扣判断**
   - 仅通过 `provider === 'coupon_free'` 判断
   - 不要通过 `pay_url === null` 判断（其他错误场景也可能没有 pay_url）

6. **倒计时展示**
   - 使用 `valid_until` 字段计算剩余有效期
   - 券即将过期（< 24h）时可以高亮提醒用户尽快使用
