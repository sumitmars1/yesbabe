# 巴西支付（PIX）前端对接文档

## 1. 概述

本文档面向前端开发者，提供巴西 PIX 支付系统的完整对接指南。

**支付提供商**: VIPPay（PIX 支付）
**货币**: BRL（巴西雷亚尔，符号 R$）
**订单号前缀**: `BR_`

**支付流程**:
```
创建订单 → 获取支付链接 → 用户扫码/支付 → 回调处理 → 权益发放
```

**支持场景**:
- VIP 订阅购买（PRO 会员）
- Token（钻石）充值

**接口路径前缀**: `/api/pay/br`

---

## 2. 鉴权说明

所有支付接口需要携带 JWT Token：

```javascript
headers: {
  "Content-Type": "application/json",
  "Authorization": "Bearer YOUR_JWT_TOKEN"
}
```

---

## 3. 支付方式

巴西支付仅支持一种支付方式：

| 值 | 说明 |
|---|---|
| `PIX` | 巴西 PIX 即时支付（扫码/复制粘贴） |

> 注意：与越南支付不同，巴西无需传 `bank_code` 等额外参数，只需传 `pay_method: "PIX"`。

## 4. VIP 订阅购买

### 4.1 创建 VIP 支付订单

**接口**: `POST /api/pay/br/create_vip`

**限流**: 10 次 / 60 秒

**请求体**:
```json
{
  "vip_id": 1,
  "pay_method": "PIX",
  "back_url": "https://example.com/payment-result",
  "coupon_id": null
}
```

**参数说明**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `vip_id` | int | 是 | VIP 套餐 ID（>0） |
| `pay_method` | string | 是 | 固定传 `"PIX"` |
| `back_url` | string | 否 | 支付完成后跳转地址，必须以 `http://` 或 `https://` 开头 |
| `coupon_id` | int | 否 | 优惠券 ID（用户优惠券实例 ID，>0） |

**成功响应**:
```json
{
  "code": 200,
  "message": "Success",
  "data": {
    "order_id": "BR_20260302120000123456100000_A1B2C3",
    "payment_url": "https://pay.vippay.pro/checkout/xxxxx"
  }
}
```

**优惠券全额抵扣响应**（无需跳转支付）:
```json
{
  "code": 200,
  "message": "Success",
  "data": {
    "order_id": "BR_20260302120000123456100000_A1B2C3",
    "payment_url": "",
    "coupon_free": true
  }
}
```

**前端处理**:
```javascript
async function createBrVipPayment(vipId, couponId = null) {
  const response = await fetch('/api/pay/br/create_vip', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${token}`
    },
    body: JSON.stringify({
      vip_id: vipId,
      pay_method: 'PIX',
      back_url: window.location.origin + '/payment-result',
      coupon_id: couponId
    })
  });

  const result = await response.json();

  if (result.code === 200) {
    // 检查是否优惠券全额抵扣
    if (result.data.coupon_free) {
      // 无需支付，权益已直接发放
      showSuccess('优惠券抵扣成功，PRO 已开通！');
      await refreshUserInfo();
      return;
    }

    // 跳转到 PIX 支付页面
    window.location.href = result.data.payment_url;
  } else {
    handlePaymentError(result);
  }
}
```

## 6. 订单查询

### 6.1 查询订单状态

**接口**: `POST /api/pay/br/query?order_id={订单号}`

**限流**: 10 次 / 60 秒

**说明**: 查询订单状态。如果订单为 `pending` 状态，后端会自动向 VIPPay 查询最新状态，如果已支付则自动补单发放权益。

> 与越南支付不同，巴西没有独立的 `sync` 接口，查询接口本身已集成补单功能。

**请求示例**:
```javascript
async function queryBrOrder(orderId) {
  const response = await fetch(
    `/api/pay/br/query?order_id=${encodeURIComponent(orderId)}`,
    {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${token}`
      }
    }
  );
  return await response.json();
}
```

**成功响应**:
```json
{
  "code": 200,
  "message": "Success",
  "data": {
    "order_id": "BR_20260302120000123456100000_A1B2C3",
    "status": "completed",
    "amount": "29.90",
    "currency": "BRL",
    "order_type": "vip_purchase",
    "created_at": "2026-03-02T12:00:00",
    "completed_at": "2026-03-02T12:05:30"
  }
}
```

### 6.2 订单状态说明

| 状态 | 说明 | 前端处理建议 |
|------|------|-------------|
| `pending` | 待支付 | 提示用户等待或重新扫码 |
| `completed` | 已完成（权益已发放） | 刷新用户信息，提示成功 |
| `failed` | 支付失败 | 提示重新发起支付 |
| `cancelled` | 已取消（超时/被替代） | 提示重新创建订单 |
| `refunded` | 已退款 | 提示订单已退款 |
| `amount_mismatch` | 金额异常（需人工审核） | 提示联系客服 |
| `benefit_failed` | 权益发放失败（需人工审核） | 提示联系客服 |

### 6.3 支付回调页面推荐实现

```javascript
/**
 * 支付完成后的回调页面逻辑
 * 用户从 PIX 支付页面跳转回来后执行
 */
async function onPaymentReturn() {
  // 从 URL 参数或本地存储获取订单号
  const orderId = getOrderIdFromUrl() || localStorage.getItem('br_last_order_id');
  if (!orderId) {
    showError('未找到订单信息');
    return;
  }

  showLoading('正在查询支付结果...');

  // 立即查询一次
  let result = await queryBrOrder(orderId);

  if (result.code === 200) {
    const { status } = result.data;

    switch (status) {
      case 'completed':
        showSuccess('支付成功，权益已发放！');
        await refreshUserInfo();
        break;

      case 'pending':
        // PIX 支付可能有几秒延迟，轮询最多 3 次
        await pollOrderStatus(orderId, 3, 5000);
        break;

      case 'failed':
        showError('支付失败，请重新尝试');
        break;

      case 'cancelled':
        showWarning('订单已取消，请重新创建');
        break;

      case 'refunded':
        showInfo('订单已退款');
        break;

      case 'amount_mismatch':
      case 'benefit_failed':
        showWarning('订单异常，请联系客服处理');
        break;

      default:
        showInfo(`订单状态: ${status}`);
    }
  } else {
    showError(result.message || '查询失败');
  }
}

/**
 * 轮询订单状态（用于 pending 状态等待支付确认）
 */
async function pollOrderStatus(orderId, maxRetries = 3, interval = 5000) {
  for (let i = 0; i < maxRetries; i++) {
    await new Promise(resolve => setTimeout(resolve, interval));

    const result = await queryBrOrder(orderId);
    if (result.code === 200 && result.data.status !== 'pending') {
      if (result.data.status === 'completed') {
        showSuccess('支付成功，权益已发放！');
        await refreshUserInfo();
      } else {
        showWarning(`订单状态: ${result.data.status}`);
      }
      return;
    }
  }

  showInfo('支付确认中，请稍后刷新页面查看');
}
```

## 7. 优惠券集成

巴西支付完整支持优惠券功能。

### 7.1 使用流程

```
获取可用优惠券列表 → 用户选择优惠券 → 创建订单时传入 coupon_id → 服务端自动计算折扣
```

### 7.2 请求示例

```javascript
// 创建 VIP 订单并使用优惠券
const result = await fetch('/api/pay/br/create_vip', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${token}`
  },
  body: JSON.stringify({
    vip_id: 2,
    pay_method: 'PIX',
    coupon_id: 42,  // 用户优惠券实例 ID
    back_url: window.location.origin + '/payment-result'
  })
});
```

### 7.3 优惠券场景说明

| 场景 | `payment_url` | `coupon_free` | 前端处理 |
|------|--------------|---------------|---------|
| 无优惠券 | 支付链接 | 无此字段 | 跳转支付 |
| 部分折扣 | 支付链接（折后价） | 无此字段 | 跳转支付 |
| 全额抵扣 | `""` 空字符串 | `true` | 无需跳转，权益已直接发放 |

### 7.4 注意事项

- 优惠券在创建订单时锁定，支付失败/取消后自动释放
- 每张优惠券只能用于一个订单
- 如果传入无效的 `coupon_id`，服务端会返回错误

## 8. 错误处理

### 8.1 常见错误码

| HTTP 状态码 | code | 说明 | 前端处理 |
|------------|------|------|---------|
| 200 | 200 | 成功 | 正常处理 |
| 400 | 400 | 请求参数错误 / 重复点击 / 套餐不存在 | 显示 `message` 提示 |
| 401 | 401 | 未登录或 Token 过期 | 跳转到登录页 |
| 404 | 404 | 订单不存在 | 提示订单不存在 |
| 422 | 422 | 请求体格式错误（Pydantic 校验失败） | 检查请求参数类型 |
| 429 | 429 | 请求过于频繁（超过 10 次/60 秒） | 提示稍后再试 |
| 500 | 500 | 服务器内部错误 | 提示系统繁忙 |

### 8.2 常见业务错误消息

| 场景 | `message` 内容 |
|------|---------------|
| 重复点击 | `A payment request is being processed, please do not click repeatedly` |
| 套餐不存在 | `Current PRO subscription method does not exist` / `Current diamond package does not exist` |
| 价格未配置 | `Price not configured for your region` |
| 支付服务不可用 | `Payment service temporarily unavailable, please try again later` |
| 订单不存在 | `Order does not exist` |

### 8.3 统一错误处理

```javascript
function handlePaymentError(result) {
  const { code, message } = result;

  switch (code) {
    case 400:
      // 参数错误、重复点击、套餐不存在等
      showError(message);
      break;
    case 401:
      showError('登录已过期，请重新登录');
      redirectToLogin();
      break;
    case 404:
      showError('订单不存在');
      break;
    case 422:
      showError('请求参数格式有误，请刷新页面重试');
      break;
    case 429:
      showError('操作过于频繁，请稍后再试');
      break;
    case 500:
    case 503:
      showError('支付服务暂时繁忙，请稍后再试');
      break;
    default:
      showError(message || '操作失败，请重试');
  }
}
```

## 9. 完整集成示例

```javascript
/**
 * 巴西 PIX 支付集成 - 完整示例
 */
class BrPaymentService {
  constructor(apiBaseUrl, getToken) {
    this.apiBase = apiBaseUrl;
    this.getToken = getToken;
    this._isProcessing = false;
  }

  /** 通用请求方法 */
  async _request(path, method = 'POST', body = null) {
    const headers = {
      'Authorization': `Bearer ${this.getToken()}`,
    };
    const options = { method, headers };

    if (body) {
      headers['Content-Type'] = 'application/json';
      options.body = JSON.stringify(body);
    }

    const response = await fetch(`${this.apiBase}${path}`, options);
    return await response.json();
  }

  /**
   * 创建 VIP 支付订单
   * @param {number} vipId - VIP 套餐 ID
   * @param {number|null} couponId - 优惠券 ID
   * @param {string|null} backUrl - 支付完成跳转地址
   */
  async createVipPayment(vipId, couponId = null, backUrl = null) {
    if (this._isProcessing) return;
    this._isProcessing = true;

    try {
      const result = await this._request('/api/pay/br/create_vip', 'POST', {
        vip_id: vipId,
        pay_method: 'PIX',
        coupon_id: couponId,
        back_url: backUrl || `${window.location.origin}/payment-result`,
      });

      if (result.code !== 200) {
        throw new PaymentError(result.code, result.message);
      }

      return result.data;
    } finally {
      // 1 秒后解除防抖锁
      setTimeout(() => { this._isProcessing = false; }, 1000);
    }
  }

  /**
   * 创建 Token 充值订单
   * @param {number} tokenId - Token 套餐 ID
   * @param {number|null} couponId - 优惠券 ID
   * @param {string|null} backUrl - 支付完成跳转地址
   */
  async createTokenPayment(tokenId, couponId = null, backUrl = null) {
    if (this._isProcessing) return;
    this._isProcessing = true;

    try {
      const result = await this._request('/api/pay/br/create_tokens', 'POST', {
        token_id: tokenId,
        pay_method: 'PIX',
        coupon_id: couponId,
        back_url: backUrl || `${window.location.origin}/payment-result`,
      });

      if (result.code !== 200) {
        throw new PaymentError(result.code, result.message);
      }

      return result.data;
    } finally {
      setTimeout(() => { this._isProcessing = false; }, 1000);
    }
  }

  /**
   * 查询订单状态（pending 状态自动补单）
   * @param {string} orderId - 订单号
   */
  async queryOrder(orderId) {
    const result = await this._request(
      `/api/pay/br/query?order_id=${encodeURIComponent(orderId)}`,
      'POST'
    );

    if (result.code !== 200) {
      throw new PaymentError(result.code, result.message);
    }

    return result.data;
  }

  /**
   * 发起支付（统一入口）
   * @param {'vip'|'token'} type - 支付类型
   * @param {number} productId - 套餐 ID
   * @param {number|null} couponId - 优惠券 ID
   */
  async pay(type, productId, couponId = null) {
    const data = type === 'vip'
      ? await this.createVipPayment(productId, couponId)
      : await this.createTokenPayment(productId, couponId);

    // 保存订单号到本地存储（用于回调页面查询）
    localStorage.setItem('br_last_order_id', data.order_id);

    // 优惠券全额抵扣，无需跳转
    if (data.coupon_free) {
      return { type: 'coupon_free', orderId: data.order_id };
    }

    // 跳转到 PIX 支付页面
    window.location.href = data.payment_url;
    return { type: 'redirect', orderId: data.order_id };
  }
}

class PaymentError extends Error {
  constructor(code, message) {
    super(message);
    this.code = code;
  }
}

// ========== 使用示例 ==========

const paymentService = new BrPaymentService('', () => getAuthToken());

// VIP 购买（不使用优惠券）
document.getElementById('btn-vip').addEventListener('click', async () => {
  try {
    const result = await paymentService.pay('vip', 1);
    if (result.type === 'coupon_free') {
      alert('优惠券全额抵扣，PRO 已开通！');
      location.reload();
    }
    // type === 'redirect' 时页面已跳转，无需额外处理
  } catch (e) {
    if (e instanceof PaymentError) {
      handlePaymentError({ code: e.code, message: e.message });
    } else {
      alert('网络错误，请检查网络连接');
    }
  }
});

// Token 充值（使用优惠券）
document.getElementById('btn-token').addEventListener('click', async () => {
  try {
    const result = await paymentService.pay('token', 3, selectedCouponId);
    if (result.type === 'coupon_free') {
      alert('优惠券全额抵扣，钻石已到账！');
      location.reload();
    }
  } catch (e) {
    if (e instanceof PaymentError) {
      handlePaymentError({ code: e.code, message: e.message });
    } else {
      alert('网络错误，请检查网络连接');
    }
  }
});
```

## 10. 与越南支付的差异对照

| 维度 | 巴西支付 (`/api/pay/br`) | 越南支付 (`/api/pay/vn`) |
|------|------------------------|------------------------|
| 支付方式 | 仅 `PIX` | VNBANKQR / VNZALO / VNMOMO / VNVTPAY |
| 货币 | BRL（巴西雷亚尔） | VND（越南盾） |
| bank_code | 不需要 | VNBANKQR 通道需要 |
| 订单号前缀 | `BR_` | `VN_` |
| 查询接口 | `POST /query` (集成补单) | `GET /query` (纯查询) + `POST /sync` (补单) |
| 取消订单 | 无独立接口（新订单自动替代旧订单） | `POST /cancel` |
| 优惠券全额抵扣 | 返回 `coupon_free: true` | 返回 `coupon_free: true` |
| 订单超时 | 30 分钟 | 30 分钟 |
| 限流 | 10 次 / 60 秒 | 10 次 / 60 秒 |

## 11. 注意事项

### 11.1 安全建议
1. 始终使用 HTTPS 传输
2. JWT Token 通过 Header 传递，不要暴露在 URL 中
3. 前端不要缓存价格信息，价格以服务端返回为准
4. 支付跳转前验证用户登录状态

### 11.2 用户体验建议
1. 支付按钮添加防抖（至少 1 秒间隔），防止重复创建订单
2. 支付跳转前显示 Loading 状态
3. 使用 `window.location.href` 跳转，不要用 `window.open`
4. 支付回调页面自动查询订单状态
5. 提供"刷新状态"按钮，应对回调延迟（PIX 通常秒到，但偶尔有延迟）
6. 对异常状态（`amount_mismatch`、`benefit_failed`）提示联系客服
7. 超时未支付的订单（30 分钟）会被自动取消
8. 每次创建新订单会自动取消该用户之前未支付的同类型订单，无需手动取消

### 11.3 常见问题

**Q1: 用户支付后跳转回来，权益未到账？**
- 调用 `/api/pay/br/query` 接口，后端会自动查询 VIPPay 并补单

**Q2: 用户多次点击支付按钮会创建多个订单吗？**
- 不会。后端有分布式锁防重复，且每次创建新订单会自动取消之前未支付的同类型订单

**Q3: 巴西用户怎么判断？**
- 后端通过用户语言（`pt` 葡萄牙语）自动路由到巴西支付，前端无需判断

**Q4: 支付链接有效期多久？**
- 订单有效期 30 分钟，过期后支付链接失效，用户需重新创建订单

**Q5: `coupon_free: true` 是什么意思？**
- 优惠券完全覆盖了订单金额，无需跳转支付，权益已直接发放。此时 `payment_url` 为空字符串

**Q6: 如何区分 VIP 订单和 Token 订单？**
- 查询接口返回的 `order_type` 字段：`vip_purchase` = VIP 订单，`token_purchase` = Token 订单
