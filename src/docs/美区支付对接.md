# YesBabe AI 前端支付对接文档

> **版本**: v2.0 | **更新日期**: 2026-03-10
>
> **适用范围**: 全区域支付（越南 VnPay/TdayPay、印度 ptmpays、巴西 VIPPay、美区 USPay、CoinGate 加密货币）

---

## 一、全局约定

### 1.1 统一响应格式

所有接口返回值均遵循以下格式：

```json
{
  "code": 200,
  "message": "Success",
  "data": { ... }
}
```

| code | 含义 |
|------|------|
| 200 | 成功 |
| 400 | 请求参数错误 / 业务逻辑错误 |
| 401 | 未认证（JWT 过期或缺失） |
| 404 | 资源不存在 |
| 429 | 请求频率过高 |
| 500 | 服务器内部错误 |

### 1.2 认证方式

所有前端调用的接口（除回调外）均需携带 JWT Token：

```
Authorization: Bearer <access_token>
```

### 1.3 区域与货币映射

| 用户语言 | 区域代码 | 货币 | 支付提供商 | 接口前缀 |
|----------|----------|------|-----------|----------|
| vi (越南语) | vi | VND | VnPay / TdayPay | `/api/pay/vn` |
| hi (印地语) | hi | INR | ptmpays | `/api/pay/hi` |
| pt (葡萄牙语) | pt | BRL | VIPPay | `/api/pay/br` |
| en (英语) | en | USD | USPay | `/api/pay/us` |
| 全语言 | global | USD | CoinGate (加密) | `/api/pay/coingate` |

> **严格区域绑定**：用户只能使用其语言区域对应的支付方式，跨区调用直接返回错误。

### 1.4 订单状态定义

| 状态 | 含义 | 是否终态 | 前端处理建议 |
|------|------|---------|-------------|
| `pending` | 待支付 | 否 | 显示"等待支付"，提供取消/重试按钮 |
| `completed` | 已完成 | 是 | 显示"支付成功"，刷新用户权益 |
| `cancelled` | 已取消 | 否（超时取消可补单） | 显示"已取消" |
| `failed` | 失败 | 是 | 显示"支付失败"，引导重新购买 |
| `amount_mismatch` | 金额异常 | 是 | 显示"订单异常，请联系客服" |
| `benefit_failed` | 权益发放失败 | 是 | 显示"支付成功但权益发放失败，请联系客服" |
| `refunded` | 已退款 | 是 | 显示"已退款" |

## 二、支付核心流程

### 2.1 标准支付时序图

```
前端                         后端                        支付平台
 │                           │                            │
 │  1. POST /create_vip      │                            │
 │  ────────────────────────► │  创建本地订单               │
 │                           │  ──────────────────────────►│
 │                           │                  返回支付链接│
 │  ◄──────────────────────── │  {order_id, payment_url}   │
 │                           │                            │
 │  2. 跳转 payment_url      │                            │
 │  ─────────────────────────────────────────────────────►│
 │                           │                  用户完成支付│
 │                           │  3. POST /notify (回调)     │
 │                           │  ◄──────────────────────────│
 │                           │  验签 → 发放权益 → "success" │
 │                           │  ──────────────────────────►│
 │                           │                            │
 │  4. 返回前端 (back_url)   │                            │
 │  ◄─────────────────────────────────────────────────────│
 │                           │                            │
 │  5. GET /query (查状态)   │                            │
 │  ────────────────────────►│                            │
 │  ◄──────────────────────── │  {status: "completed"}     │
 │                           │                            │
 │  [可选] POST /sync (补单)  │                            │
 │  ────────────────────────►│  查询远端 → 核金额 → 发权益 │
 │  ◄──────────────────────── │  {synced: true}            │
```

### 2.2 优惠券全免流程

当优惠券折扣 >= 订单金额时，不跳转支付平台，直接完成：

```
前端                         后端
 │                           │
 │  POST /create_vip         │
 │  {coupon_id: 42}          │
 │  ────────────────────────►│  验证优惠券 → final_amount ≤ 0
 │                           │  直接标记 completed + 发放权益
 │  ◄──────────────────────── │  {order_id, payment_url: "", coupon_free: true}
 │                           │
 │  检测 coupon_free=true    │
 │  直接显示"支付成功"        │
```

## 三、各区域支付接口详情

### 3.1 越南支付 (VnPay / TdayPay)

**接口前缀**: `/api/pay/vn`

#### 3.1.1 创建VIP订阅订单

```
POST /api/pay/vn/create_vip
```

**请求体**:

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| vip_id | int | 是 | VIP 套餐 ID (> 0) |
| pay_method | string | 是 | 支付方式，见下方枚举 |
| coupon_id | int | 否 | 优惠券实例 ID (> 0) |
| back_url | string | 否 | 支付后跳转 URL (≤ 256 字符，需 http/https 开头) |
| bank_code | string | 否 | 银行代码，仅 `VNBANKQR` 时有效 |

**pay_method 枚举值**:

| 值 | 说明 |
|----|------|
| VNBANKQR | 越南银行二维码 |
| VNZALO | Zalo Pay |
| VNMOMO | Momo 支付 |
| VNVTPAY | ViettelPay |
| VNBANKQR2 | 银行二维码 (备用通道) |
| VNMO2MO | Momo (备用通道) |
| VNZA2LO | Zalo Pay (备用通道) |
| VNVT2PAY | ViettelPay (备用通道) |

**bank_code 枚举值** (仅 `VNBANKQR` 时):
`ICB`, `MB`, `TCB`, `ACB`, `VPB`, `MSB`, `BAB`, `VAB`, `VIETBANK`, `SEAB`, `VCB`, `TPB`, `NAB`

**成功响应**:

```json
{
  "code": 200,
  "message": "Success",
  "data": {
    "order_id": "VN20260310153045123456",
    "payment_url": "https://pay.vnpay.vn/...",
    "coupon_free": false
  }
}
```

#### 3.1.2 创建Token充值订单

```
POST /api/pay/vn/create_tokens
```

**请求体**:

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| token_id | int | 是 | Token 套餐 ID (> 0) |
| pay_method | string | 是 | 同 VIP 支付方式 |

### 4.4 支付时使用优惠券

在创建订单时传入 `coupon_id`：

```json
{
  "vip_id": 1,
  "pay_method": "VNMOMO",
  "coupon_id": 42
}
```

**后端处理逻辑**:
- 验证优惠券是否属于当前用户、未过期、未使用
- 验证适用范围（VIP/Token/全部）
- 计算折扣：`final_amount = amount - discount`
- 如果 `final_amount ≤ 0`：直接完成订单，返回 `coupon_free: true`
- 如果 `final_amount > 0`：以 `final_amount` 发起支付

## 五、前端关键处理逻辑

### 5.1 创建订单后的处理

```typescript
async function createPayment(params) {
  const res = await api.post('/api/pay/{region}/create_vip', params);

  if (res.code !== 200) {
    // 显示错误信息（已 i18n）
    showError(res.message);
    return;
  }

  if (res.data.coupon_free) {
    // 优惠券全免，无需跳转支付
    showSuccess('支付成功');
    refreshUserProfile();
    return;
  }

  // 跳转到支付页面
  window.location.href = res.data.payment_url;
}
```

### 5.2 支付返回后的状态轮询

用户从支付页面返回后（通过 `back_url`），应轮询订单状态：

```typescript
async function pollOrderStatus(orderId: string, region: string) {
  const paramKey = region === 'coingate' ? 'payment_id' : 'mch_order_id';
  const maxAttempts = 10;  // 最多轮询 10 次
  const interval = 3000;   // 每 3 秒

  for (let i = 0; i < maxAttempts; i++) {
    const res = await api.get(`/api/pay/${region}/query?${paramKey}=${orderId}`);

    if (res.data.local_status === 'completed') {
      showSuccess('支付成功');
      refreshUserProfile();
      return;
    }

    if (['amount_mismatch', 'benefit_failed', 'refunded'].includes(res.data.local_status)) {
      showError('订单异常，请联系客服');
      return;
    }

    await sleep(interval);
  }

  // 轮询超时，提供手动补单按钮
  showSyncButton(orderId, region);
}
```

### 5.3 手动补单 (sync)

当回调丢失时，用户点击"确认支付"触发补单：

```typescript
async function syncOrder(orderId: string, region: string) {
  const paramKey = region === 'coingate' ? 'payment_id' : 'mch_order_id';
  const res = await api.post(`/api/pay/${region}/sync?${paramKey}=${orderId}`);

  if (res.code === 200 && res.data.synced) {
    showSuccess(res.data.message);
    refreshUserProfile();
  } else {
    showInfo(res.data.message);
  }
}
```

### 5.4 防重复点击

后端已有分布式锁保护（120 秒内同一用户不能重复创建同类订单），前端也应做按钮防抖：

```typescript
// 支付按钮点击后 disable 3 秒
payButton.disabled = true;
setTimeout(() => payButton.disabled = false, 3000);
```

如果用户重复点击，后端返回：
```json
{
  "code": 400,
  "message": "A payment request is being processed, please do not click repeatedly"
}
```

## 九、注意事项

1. **query 与 sync 的区别**：
   - `query` = 纯只读，不触发任何业务逻辑，用于前端轮询显示状态
   - `sync` = 查询远端 + 金额验证 + 发放权益，用于回调丢失时的补单操作

2. **back_url 的作用**：用户在支付平台完成支付后自动跳回的页面 URL，建议传应用内的订单详情页地址。

3. **订单有效期**：各区域默认 15 分钟。超时后系统自动取消 pending 订单并释放优惠券。用户重新发起支付会创建新订单。

4. **金额不可修改**：`final_amount`（优惠后应付金额）在创建订单时确定，回调和补单均不会覆盖。

5. **CoinGate 特殊性**：
   - 参数名为 `payment_id`（非 `mch_order_id`）
   - 不限区域，所有语言的用户均可使用
   - 支持 BTC、ETH、USDT 等多种加密货币，用户在 CoinGate 页面选择
