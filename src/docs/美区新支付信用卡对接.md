# HiEasy 信用卡支付 前端对接文档

> **版本**: v1.1 | **更新日期**: 2026-03-12
>
> **适用范围**: 美区（en）用户 — HiEasy 信用卡（CARD）支付通道
>
> **接口前缀**: `/api/pay/hieasy`
>
> **与旧 USPay 的关系**: HiEasy 作为美区第二支付通道（信用卡），与旧 USPay (`/api/pay/us`) 并存。

---

## 一、全局约定

### 1.1 统一响应格式

```json
{
  "code": 200,
  "message": "Success",
  "data": { ... }
}
```

| code | 含义 |
|------|------|
| 200 | 成功 |
| 400 | 请求参数错误 / 业务逻辑错误 |
| 401 | 未认证（JWT 过期或缺失） |
| 404 | 资源不存在 |
| 429 | 请求频率过高 |
| 500 | 服务器内部错误 |

### 1.2 认证方式

所有前端调用的接口均需携带 JWT Token：

```
Authorization: Bearer <access_token>
```

### 1.3 区域限制

- **仅限英语区域（en）用户**使用，非 en 用户调用直接返回错误
- 货币固定为 **USD**
- 支付方式固定为 **CARD**（信用卡/借记卡）

### 1.4 订单状态定义

| 状态 | 含义 | 是否终态 | 前端处理建议 |
|------|------|---------|-------------|
| `pending` | 待支付 | 否 | 显示"等待支付"，提供取消/重试按钮 |
| `completed` | 已完成 | 是 | 显示"支付成功"，刷新用户权益 |
| `cancelled` | 已取消 | 否（超时取消可补单） | 显示"已取消" |
| `failed` | 失败 | 是 | 显示"支付失败"，引导重新购买 |
| `amount_mismatch` | 金额异常 | 是 | 显示"订单异常，请联系客服" |
| `benefit_failed` | 权益发放失败 | 是 | 显示"支付成功但权益发放失败，请联系客服" |
| `refunded` | 已退款 | 是 | 显示"已退款" |

## 二、支付核心流程

### 2.1 标准支付时序图

```
前端                         后端                        HiEasy
 │                           │                            │
 │  1. POST /create_vip      │                            │
 │  ────────────────────────► │  创建本地订单               │
 │                           │  ──────────────────────────►│
 │                           │              返回 payData   │
 │  ◄──────────────────────── │  {order_id, payment_url}   │
 │                           │                            │
 │  2. 跳转 payment_url      │                            │
 │  ─────────────────────────────────────────────────────►│
 │                           │          用户完成信用卡支付  │
 │                           │  3. POST /notify (回调)     │
 │                           │  ◄──────────────────────────│
 │                           │  验签 → 发放权益 → "success" │
 │                           │  ──────────────────────────►│
 │                           │                            │
 │  4. 返回前端 (back_url)   │                            │
 │  ◄─────────────────────────────────────────────────────│
 │                           │                            │
 │  5. GET /query (查状态)   │                            │
 │  ────────────────────────►│                            │
 │  ◄──────────────────────── │  {status: "completed"}     │
 │                           │                            │
 │  [可选] POST /sync (补单)  │                            │
 │  ────────────────────────►│  查询远端 → 核金额 → 发权益 │
 │  ◄──────────────────────── │  {synced: true}            │
```

### 2.2 优惠券全免流程

当优惠券折扣 >= 订单金额时，不跳转支付页面，直接完成：

```
前端                         后端
 │                           │
 │  POST /create_vip         │
 │  {coupon_id: 42}          │
 │  ────────────────────────►│  验证优惠券 → final_amount ≤ 0
 │                           │  直接标记 completed + 发放权益
 │  ◄──────────────────────── │  {order_id, payment_url: "", coupon_free: true}
 │                           │
 │  检测 coupon_free=true    │
 │  直接显示"支付成功"        │
```

## 三、接口详情

### 3.1 创建 VIP 订阅订单

```
POST /api/pay/hieasy/create_vip
```

**请求体**:

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| vip_id | int | 是 | VIP 套餐 ID (> 0) |
| pay_method | string | 是 | 固定传 `"CARD"` |
| coupon_id | int | 否 | 优惠券实例 ID (> 0) |
| back_url | string | 否 | 支付完成后跳转 URL (≤ 256 字符，需 http/https 开头) |

**成功响应**:

```json
{
  "code": 200,
  "message": "Success",
  "data": {
    "order_id": "HE_20260312153045_a1b2c3d4",
    "payment_url": "https://payment.hieasy.vip/checkout/...",
    "coupon_free": false
  }
}
```

**优惠券全免响应** (`coupon_free: true`):

```json
{
  "code": 200,
  "message": "Success",
  "data": {
    "order_id": "HE_20260312153045_a1b2c3d4",
    "payment_url": "",
    "coupon_free": true
  }
}
```

### 3.2 创建 Token 充值订单

```
POST /api/pay/hieasy/create_tokens
```

**请求体**:

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| token_id | int | 是 | Token 套餐 ID (> 0) |
| pay_method | string | 是 | 固定传 `"CARD"` |
| coupon_id | int | 否 | 优惠券实例 ID (> 0) |
| back_url | string | 否 | 支付完成后跳转 URL (≤ 256 字符，需 http/https 开头) |

**响应格式**: 同 VIP 订阅

### 3.3 查询订单状态（纯只读）

```
GET /api/pay/hieasy/query?mch_order_id=HE_20260312153045_a1b2c3d4
```

**查询参数**:

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| mch_order_id | string | 是 | 商户订单号 |

**响应**:

```json
{
  "code": 200,
  "message": "Success",
  "data": {
    "order_id": "HE_20260312153045_a1b2c3d4",
    "local_status": "completed",
    "remote_status": "2",
    "remote_status_desc": "支付成功",
    "amount": 9.99,
    "currency": "USD",
    "order_type": "vip_purchase",
    "created_at": "2026-03-12T15:30:45",
    "completed_at": "2026-03-12T15:32:10"
  }
}
```

**remote_status 含义** (HiEasy 平台状态码):

| 值 | 说明 |
|----|------|
| 0 | 订单生成 |
| 1 | 支付中 |
| 2 | 支付成功 |
| 3 | 支付失败 |
| 4 | 已撤销 |
| 5 | 已退款 |
| 6 | 订单关闭 |

### 3.4 同步订单（补单）

当回调丢失时，用户可手动触发补单。后端主动向 HiEasy 查询订单状态，如果已支付则自动发放权益。

```
POST /api/pay/hieasy/sync?mch_order_id=HE_20260312153045_a1b2c3d4
```

**查询参数**:

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| mch_order_id | string | 是 | 商户订单号 |

**响应**:

```json
{
  "code": 200,
  "message": "Success",
  "data": {
    "order_id": "HE_20260312153045_a1b2c3d4",
    "synced": true,
    "previous_status": "pending",
    "current_status": "completed",
    "message": "Payment confirmed and benefits granted"
  }
}
```

**补单限制**:
- 已完成（`completed`）的订单返回 "Order already completed"
- 异常状态（`amount_mismatch` / `benefit_failed` / `refunded`）的订单返回 "Order has issues, please contact support"
- 用户主动取消的订单不可补单
- 被新订单替代的订单不可补单

## 四、前端关键处理逻辑

### 4.1 创建订单后的处理

```typescript
async function createHieasyPayment(params: {
  vip_id?: number;
  token_id?: number;
  coupon_id?: number;
  back_url?: string;
}) {
  // 根据购买类型选择端点
  const endpoint = params.vip_id
    ? '/api/pay/hieasy/create_vip'
    : '/api/pay/hieasy/create_tokens';

  const body = {
    ...(params.vip_id ? { vip_id: params.vip_id } : { token_id: params.token_id }),
    pay_method: 'CARD',  // 固定值：信用卡支付
    coupon_id: params.coupon_id,
    back_url: params.back_url || window.location.href,
  };

  const res = await api.post(endpoint, body);

  if (res.code !== 200) {
    showError(res.message);  // 错误信息已 i18n
    return;
  }

  if (res.data.coupon_free) {
    // 优惠券全免，无需跳转支付
    showSuccess('Payment successful');
    refreshUserProfile();
    return;
  }

  // 保存 order_id 用于返回后查询
  localStorage.setItem('pending_order_id', res.data.order_id);

  // 跳转到信用卡支付页面
  window.location.href = res.data.payment_url;
}
```

### 4.2 支付返回后的状态轮询

用户从支付页面返回后（通过 `back_url`），应轮询订单状态：

```typescript
async function pollHieasyOrderStatus(orderId: string) {
  const maxAttempts = 10;  // 最多轮询 10 次
  const interval = 3000;   // 每 3 秒

  for (let i = 0; i < maxAttempts; i++) {
    const res = await api.get(`/api/pay/hieasy/query?mch_order_id=${orderId}`);

    if (res.data.local_status === 'completed') {
      showSuccess('Payment successful');
      refreshUserProfile();
      return;
    }

    if (['failed', 'amount_mismatch', 'benefit_failed', 'refunded'].includes(res.data.local_status)) {
      showError('Order issue, please contact support');
      return;
    }

    await sleep(interval);
  }

  // 轮询超时，提供手动补单按钮
  showSyncButton(orderId);
}
```

### 4.3 手动补单 (sync)

当回调丢失时，用户点击"确认支付"触发补单：

```typescript
async function syncHieasyOrder(orderId: string) {
  const res = await api.post(`/api/pay/hieasy/sync?mch_order_id=${orderId}`);

  if (res.code === 200 && res.data.synced) {
    showSuccess(res.data.message);
    refreshUserProfile();
  } else {
    showInfo(res.data.message);
  }
}
```

### 4.4 防重复点击

后端已有分布式锁保护（120 秒内同一用户不能重复创建同类订单），前端也应做按钮防抖：

```typescript
// 支付按钮点击后 disable 3 秒
payButton.disabled = true;
setTimeout(() => payButton.disabled = false, 3000);
```

如果用户重复点击，后端返回：
```json
{
  "code": 400,
  "message": "A payment request is being processed, please do not click repeatedly"
}
```

## 五、支付方式

| pay_method | 说明 | 备注 |
|-----------|------|------|
| CARD | 信用卡/借记卡 | 当前唯一的支付方式 |

> HiEasy 平台支持多种 payWay（KAKAOPAY / PIX / UPI / PAYPAL 等），但当前商户号仅开通了 **CARD**（信用卡）。前端固定传 `"CARD"` 即可，后端会使用配置中心的 `HIEASY_PAY_WAY` 值覆盖。

## 七、常见错误信息

| 场景 | message (i18n) | 处理建议 |
|------|---------------|---------|
| 非美区用户 | This payment method is not available for your region | 检查用户语言设置是否为 en |
| 价格未配置 | Price not configured for your region | 联系运营配置 en 语言的 i18n 价格 |
| 套餐不存在 | Current PRO subscription method does not exist | 刷新套餐列表 |

| 支付未完成 | Payment not yet completed | 继续等待或手动补单 |
| 订单已取消 | Order was cancelled by user, please contact support for refund | 引导用户联系客服 |
| 订单已被替代 | Order was replaced by a new order | 使用新订单 |

## 九、注意事项

1. **query 与 sync 的区别**：
   - `query` = 纯只读，不触发任何业务逻辑，用于前端轮询显示状态
   - `sync` = 查询远端 + 金额验证 + 发放权益，用于回调丢失时的补单操作

2. **back_url 的作用**：用户在支付页面完成支付后自动跳回的页面 URL，建议传应用内的订单详情页地址。

3. **订单有效期**：HiEasy 通道默认 **60 分钟**（较其他通道更长）。超时后前端可提示用户重新发起支付。

4. **金额不可修改**：`final_amount`（优惠后应付金额）在创建订单时确定，回调和补单均不会覆盖。

5. **pay_method 值**：前端传 `"CARD"`。后端实际使用 `settings.HIEASY_PAY_WAY` 配置值发送给 HiEasy API，不依赖前端传值，确保安全。

6. **熔断保护**：HiEasy API 有熔断器保护（5 次 5xx/网络错误触发，60 秒冷却期）。熔断期间创建订单会返回 "Payment service temporarily unavailable"，前端应提示用户稍后重试。
