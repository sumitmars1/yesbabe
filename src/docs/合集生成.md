# 🧩 聊天框合集功能逻辑说明

以下为 **聊天框合集功能** 的完整交互与处理逻辑说明。
---

## 一、功能目标

当接收到合集营销信息（`collection_marketing`）时，需要：

1. 展示合集的基本信息（标题、价格、图片数量、视频数量、封面图等）。  
2. 对封面图添加 **打码遮罩（模糊/像素化）**，遮盖整个图片。  
3. 在封面图上添加「购买按钮」。  
4. 点击按钮后发送购买请求。  
5. 购买成功后展示合集中的图片与视频内容。

---

## 二、数据交互流程

### 1️⃣ 接收合集营销信息

**接口：** `api/ws/chat`  
**类型：** `collection_marketing`

**示例：**
```json
{
	"type": "collection_marketing",
	"data": {
		"id": 1,
		"title": "测试合集",
		"price": 400,
		"image_count": 20,
		"video_count": 2,
		"cover_image": "ff6302cf-2a3d-49f4-b21b-244a339bbbff_0"
	},
	"timestamp": "2025-10-24T14:44:59.556659"
}
````

**前端行为：**

* 显示：

  * 标题（`title`）
  * 价格（`price`）
  * 图片数量（`image_count`）
  * 视频数量（`video_count`）
* 对 `cover_image` 添加全屏遮罩打码效果（模糊或像素化）。
* 在封面图中央显示「购买合集」按钮。
* 点击「购买合集」按钮后，触发购买请求逻辑（见下一节）。

---

### 2️⃣ 发送购买请求

**接口：** `api/ws/chat`
**类型：** `purchase_collection`

**格式：**

```json
{
    "type": "purchase_collection",
    "collection_id": 1
}
```

> ⚠️ `collection_id` 为合集的唯一标识，点击购买按钮后发送此消息。

**前端行为：**

* 点击按钮后立即禁用按钮或显示加载状态，防止重复点击。
* 将 `collection_id` 填入请求体发送到 WebSocket。
* 等待服务端返回 `collection_purchase_success` 消息。

---

### 3️⃣ 接收购买成功信息

**类型：** `collection_purchase_success`

#### 图片合集示例

```json
{
	"type": "collection_purchase_success",
	"data": {
		"type": "collection_image",
		"message_id": 2203,
		"response_id": 2154,
		"file_count": 2,
		"content": "9c2f1ca3-884f-4d0f-ad9d-d59ab433a94a_0|ff6302cf-2a3d-49f4-b21b-244a339bbbff_0"
	},
	"timestamp": "2025-10-24T14:48:42.912993"
}
```

#### 视频合集示例

```json
{
	"type": "collection_purchase_success",
	"data": {
		"type": "collection_video",
		"message_id": 2204,
		"response_id": 2155,
		"file_count": 2,
		"content": "8326b6ae-580e-4d3f-9a8f-628487c96d5b|1a347c30-be6f-4de0-a284-f85aeb118758"
	},
	"timestamp": "2025-10-24T14:48:42.914077"
}
```

**前端处理逻辑：**

* 判断 `data.type`：

  * 若为 `collection_image` → 展示图片合集
  * 若为 `collection_video` → 展示视频合集
* 将 `content` 使用 `|` 分隔为文件名数组。
* 使用 **合集文件查询接口** 获取实际内容（见第 5 节）。
* 在购买成功后移除封面打码效果，并更新按钮状态为「已购买」。

---

### 4️⃣ 历史消息中的合集记录

**接口：** `/api/message/history?companion_id=2&offset=0&limit=20`

**示例：**

```json
{
	"code": 200,
	"message": "ok",
	"data": {
		"items": [
			{
				"id": 2203,
				"type": "collection_image",
				"response_type": "collection_image",
				"response_content": "9c2f1ca3-884f-4d0f-ad9d-d59ab433a94a_0|ff6302cf-2a3d-49f4-b21b-244a339bbbff_0"
			},
			{
				"id": 2204,
				"type": "collection_video",
				"response_type": "collection_video",
				"response_content": "8326b6ae-580e-4d3f-9a8f-628487c96d5b|1a347c30-be6f-4de0-a284-f85aeb118758"
			}
		]
	}
}
```

**前端处理：**

* 当历史消息中存在合集类型（`collection_image` / `collection_video`）时：

  * 解析 `response_content`。
  * 调用文件查询接口获取实际内容进行展示。
  * 确保历史展示逻辑与实时展示一致。

---

### 5️⃣ 合集文件查询接口

**接口：**

```
api/gen_image/collection/?filename={文件名}
```

**说明：**

* `filename` 来源于 `content` 或 `response_content`。
* 接口返回值为 **Blob 流**（图片或视频）。
* 前端需根据文件类型（图片 / 视频）进行展示。
* 获取方式与单张图片查询接口一致，可通过 `URL.createObjectURL(blob)` 生成可展示 URL。

**示例：**

```bash
GET api/gen_image/collection/?filename=2799d855-42d4-47a5-8fa1-a8035d60c913_0
```

---

## 三、整体交互流程摘要

| 步骤 | 操作说明      | 类型 / 接口                       | 前端行为             |
| -- | --------- | ----------------------------- | ---------------- |
| 1  | 接收合集营销信息  | `collection_marketing`        | 展示封面、打码、购买按钮     |
| 2  | 用户点击购买    | `purchase_collection`         | 发送购买请求，禁用按钮      |
| 3  | 服务端返回购买成功 | `collection_purchase_success` | 调用文件接口展示内容       |
| 4  | 查看历史合集    | `/api/message/history`        | 还原合集展示内容         |
| 5  | 获取合集文件    | `api/gen_image/collection`    | 获取 Blob 流显示图片/视频 |

---

## 四、视觉与交互要求

* **封面遮罩：**

  * 遮罩应覆盖整个封面图片。
  * 支持模糊（blur）或像素化（mosaic）效果。
* **购买按钮：**

  * 位于封面中央。
  * 有悬停与点击反馈。
  * 购买后按钮状态更新为「已购买」。
* **成功展示：**

  * 移除封面遮罩。
  * 显示合集的图片与视频内容。
  * 支持预览与点击查看大图/播放视频。

---

## 五、开发注意事项

* `collection_marketing` → 展示合集信息（含打码与购买按钮）
* `purchase_collection` → 用户发起购买请求
* `collection_purchase_success` → 解析并加载合集内容
* `api/gen_image/collection` → 获取文件实际数据（Blob）
* 历史记录中通过 `response_type` 与 `response_content` 可还原展示

---
## 六、总结

该合集功能主要实现流程如下：

1. **接收合集营销消息** → 显示带打码封面
2. **点击购买合集按钮** → 向服务器发送购买请求
3. **接收购买成功消息** → 调用文件接口加载合集内容
4. **展示图片与视频合集内容** → 支持历史记录恢复
5. **文件加载** 通过 `api/gen_image/collection` 接口完成
